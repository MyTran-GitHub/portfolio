THE CONSTRAINT

A quarter-century of Landsat over 1,000,000 km² exceeds any single machine's memory. The naïve pipeline dies at the boundary — the engineering is in never loading it all at once.

Nothing downstream ever opens a raw scene file — every stage after S3 talks to metadata or arrays, never paths.

Three choices, and why the alternative loses

STAC > FTP

FTP makes you know the filename in advance. STAC lets you query by geometry and date — the catalog does the indexing, not a folder-naming convention.

Xarray > GDAL scripts

Hand-rolled GDAL loops force you to track tiling and reprojection yourself. Xarray gives labeled, lazy N-D arrays — Dask handles the chunking underneath.

Embeddings > NDVI

A hand-picked index like NDVI throws away everything it wasn't designed to see. A foundation-model embedding keeps the full spectral-temporal signal for later causal work.

# THE MODERN EO STACK · LAZY-LOADED FM INFERENCE
# 1. SpatioTemporal Discovery
catalog = pystac_client.Client.open("https://landsatlook.usgs.gov/stac-browser")
items = catalog.search(bbox=[...], datetime="2000/2025",
                        collections=["landsat-c2-l2"]).item_collection()

# 2. Lazy "data cube" construction
# No data is read until .compute() is called
cube = stackstac.stack(items, epsg=4326, resolution=30, chunksize=2048)

# 3. Distributed inference (foundation model)
embeddings = cube.groupby("time").map(lambda x: fm_encoder.predict(x))

# 4. Serialize to columnar feature store
embeddings.to_geoparquet("planetary_archive.parquet", compression="snappy")

Engineering for causal truth

None of this pipeline is the finding — it's the precondition for one. A synthetic-control estimate is only as trustworthy as the embeddings it matches donors on.

A hand-picked index compresses reality before a causal model ever sees it, hiding the confounders it can't represent. A high-fidelity embedding, computed at planetary scale and re-derivable on demand, is what makes a downstream causal estimate defensible rather than convenient.
