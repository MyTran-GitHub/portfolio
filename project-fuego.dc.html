The sub-minute rerun

A responder changing a fire date shouldn't wait on a federal API. The trade-off that makes re-runs fast: cache the terrain once, recompute only what actually changes.

Terrain & fuel rasters

LANDFIRE fuels, elevation, slope, aspect — location-dependent only. Downloaded once, reused across every weather scenario on that AOI.

DYNAMIC · sim_slug
Weather & ignition

Date, start hour, wind, ignition point — recomputed per run. Cheap to regenerate; this is the only part a "rerun" actually reruns.

340,000 cells → 8KB of GeoJSON

Cell2Fire writes dense CSV burn grids in EPSG:5070. Shipping that to a browser would kill frame rate — so every timestep gets unioned, simplified, and reprojected before it ever reaches Mapbox.

RASTER → GEOJSON · src/vectorizer.py
# Convert one Cell2Fire burn-grid timestep into a Mapbox-ready polygon
def vectorize_timestep(burn_grid, transform, src_crs):
    # Extract burned cells as raw polygons (rasterio.features)
    shapes = features.shapes(burn_grid, transform=transform)
    polygons = [shape(geom) for geom, val in shapes if val == 1]

    # Union + smooth to remove blocky grid-cell artifacts
    merged = unary_union(polygons).simplify(tolerance=1.5)

    # EPSG:5070 → WGS84, truncate to ~1m precision
    wgs84 = reproject(merged, src_crs, "EPSG:4326")
    return truncate_coords(wgs84, precision=5)
